# Customer-Churn-Prediction-Project

End to End churn Prediction model built using python libraries. Compared different ML Models and analyzed Recall and Precision values.
Things Explored:
1. Feauture Importance
2. Churn risk segmentation
3. Grouped Analysis
4. Revenue analysis

“I analyzed feature importance and customer segments to identify churn drivers. The key factors were short tenure, month-to-month contracts, and higher monthly charges. For example, customers on flexible contracts had significantly higher churn compared to long-term users.
This suggests that lack of commitment and pricing sensitivity are major reasons for churn. Based on this, I would recommend targeting high-risk customers with personalized offers, promoting long-term subscriptions, and improving early-stage customer engagement to reduce churn.”



###### “The model achieved ~78% accuracy, but more importantly, recall for churn class was ~48%, indicating that nearly half of potential churners were identified. This highlights scope for improving recall since missing churners directly impacts revenue.”
###### “While precision was ~62%, meaning predictions were reasonably reliable, the lower recall suggests that improving churn detection should be prioritized over overall accuracy.”
